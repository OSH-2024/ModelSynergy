### 预期成果/目标
这个倒不是调研报告的内容，不过我想借此机会讨论一下。
首先我觉得我们做出来的简单个人助理应该不涉及图形化界面的理解问题，这个太难了。也就是说个人助理所做的所有事情都是**理论上程序员可以使用命令行完成的工作**。我建议可以以OS作业的相关内容作为一部分目标。
我这里列举了我设想的一些目标，也就是我们的简单个人助理能从自然语言中理解这些任务并执行。部分想法来源于目前的两次实验。后续实验布置后也可以参考。
打勾（[x]）或者删除（~~该条目~~）来表达你的意见。当然，可以添加新的条目。
- [ ] 一些基本的Linux操作，比如切换账号、登出、切换工作目录等等。
- [ ] 文件系统的基本操作。比如根据文件名查找，移动文件位置，删除文件。
- [ ] 创建一个文本文件/代码文件。这一点单独提出，因为文件的具体内容是大模型自己生成的。运用大模型的文本生成和代码能力。
- [ ] Git 相关操作。
- [ ] 从一些众所周知的网站上下载文件。
- [ ] 添加系统调用。
- [ ] 裁剪Linux内核。这个恐怕很有难度。这还涉及到大模型能否理解类似于menuconfig这样的界面。
